# GitHubチュートリアル

## はじめに
　GitHubは，プログラムのソースコードや文書などのファイルを保存し，変更履歴を管理できるWebサービスです．個人での開発だけでなく，複数人での共同開発，課題の管理，ソースコードのレビュー，ソフトウェアの公開などにも利用されています． <br>

　GitHubの中心となっているのが，**Git**と呼ばれるバージョン管理システムです．Gitを利用すると，「いつ，誰が，どのファイルを，どのように変更したか」を記録できます．そのため，過去の状態を確認したり，問題が発生する前の状態に戻したりできます．GitHubは，Gitで管理しているプロジェクトをインターネット上に保存し，他の利用者と共有できるようにしたプラットフォームです． <br>

　例えば，プログラムを誤って変更して動作しなくなった場合でも，Gitに変更履歴が残っていれば，以前の状態と比較して修正できます．また，複数人が同じプロジェクトを編集する場合には，それぞれの変更内容を確認しながら統合できます． <br>

　このチュートリアルでは，GitHubを初めて利用する人を対象として，GitとGitHubの基本的な概念から，リポジトリの作成，ファイルの更新，変更履歴の記録，GitHubへのアップロードまでを順番に説明します．

## 事前に準備するもの

チュートリアルを進めるために，次のものを準備します．

* GitHubアカウント
* GitがインストールされたPC
* テキストエディタまたはVisual Studio Code
* インターネット接続
* ターミナルを操作できる環境

GitHubを利用するには個人用アカウントが必要です．アカウントの作成後は，登録したメールアドレスの確認を行います．

Windows環境では，PowerShell，コマンドプロンプト，Git Bash，WSLなどからGitを操作できます．本チュートリアルでは，ターミナルからコマンドを入力する方法を中心に説明します．
